Help & features

LingoCard is an app for creating and printing flashcards for German verbs. The goal is simple: less manual work, more solid cards, and PDF exports without surprises.

01 What is a card?
basics

A card is one verb and its related data:

  • translations and context
  • verb forms (Präsens / Präteritum / Partizip II / auxiliary)
  • examples (DE/RU)
  • government/valency (Rektion), tags, frequency

A card is a structure. The look can change — the meaning stays.

02 Left list and right list
workflow

The app has two card lists.

Left list — the main/reference list, typically used for printing and export.

Right list — the draft area. It’s a safe place to try, expand, and fix cards without risking the main set.

Idea: drafts exist so a mistake doesn’t turn into a disaster.
03 Creating and editing cards
practice

You can create and edit cards manually. You can also use AI (if connected).

  • Create new cards in the draft list first if you want to review the result.
  • Move a card to the main list when it looks correct.

Best approach: get the meaning right first, then polish the layout.

04 Blocks and layout
layout

A card consists of blocks (translations, forms, examples, etc.).

Typically you can:

  • show/hide blocks
  • drag and resize blocks
  • fit the card to the print format

The idea: build the card like a constructor kit, tailored to your format.

05 PDF export and printing
print

LingoCard can generate a PDF for printing.

  • The preview should match the final PDF.
  • If something doesn’t fit, you’ll see it before exporting.

Tip: perfect 1–2 cards first, then export in bulk.

06 Local AI: LM Studio (install & setup)
AI

LM Studio is an app for running language models locally on your PC.

Official website: https://lmstudio.ai

Getting started:

  1. Download LM Studio from the official website and install it.
  2. Launch LM Studio and download a model (without a model it won’t run — that’s normal).
  3. Open Local Server and start the server (Start Server).

Recommended model settings:

  • Type: Chat / Instruct (not Base)
  • Size: 7B–8B (quality/speed balance)
  • Format: GGUF
  • Quantization: Q4_K_M or Q5_K_M

Connect to LingoCard:

In the AI Control Panel, set the endpoint to:

http://localhost:1234/v1
Important: if LM Studio isn’t running, the app still works — just without AI. AI is a helper, not a requirement.

If it’s slow: use a smaller model, close heavy apps, and check the server is running.

07 Ways to enter verbs for generation
AI

You can feed verbs into the generator in several ways — from a single infinitive to a full batch.

1) One infinitive (the “Infinitive” field)

  • Type an infinitive and press Generate.
  • Autocomplete: Tab or → accepts the suggestion.
New mode: you can type the verb in Russian or English. The panel will try to translate it into a German infinitive (via LM Studio) and then generate the card as usual.

2) Multiple infinitives in the same field

You can enter multiple verbs at once:

  • comma-separated: gehen, kommen, bleiben
  • space-separated: gehen kommen bleiben
  • multi-word verbs in quotes: "Rad fahren" "spazieren gehen"
Important: spaces are treated as separators. Use quotes for multi-word verbs.

3) Batch list via file (“Import verbs from file”)

Best for mass generation.

  • Recommended format: 1 infinitive per line.
  • Comma/space lists may work, but lines are more reliable.

4) Import cards via JSON (no generation)

If you already have ready-made cards in JSON, you can import them into the app. This is data import, not AI generation.

  • Use import when the cards are already filled.
  • If a card is incomplete, improve it in the right draft list first, then move it to the left list.

Recommended workflow

  1. Generate → the card appears in the right list
  2. Review/improve → move to the left list
  3. Export PDF
08 Data safety & what to do if things feel “off”
calm mode

If the app behaves strangely:

  • don’t delete everything immediately
  • check which list you’re working in (main / draft)
  • check whether a mode is enabled that hides/filters items

Best strategy: small changes and frequent result checks.
